add_event(location_name: str, item_name: str | None = None, rule: CollectionRule | Rule[Any] | None = None, location_type: type[Location] | None = None, item_type: type[Item] | None = None, show_in_spoiler: bool = True) → Item¶

Adds an event location/item pair to the region.

Parameters:

  • location_name – Name for the event location.

  • item_name – Name for the event item. If not provided, defaults to location_name.

  • rule – Callable to determine access for this event location within its region.

  • location_type – Location class to create the event location with. Defaults to BaseClasses.Location.

  • item_type – Item class to create the event item with. Defaults to BaseClasses.Item.

  • show_in_spoiler – Will be passed along to the created event Location’s show_in_spoiler attribute.

Returns:

The created Event Item

add_exits(exits: Iterable[str] | Mapping[str, str | None], rules: Mapping[str, CollectionRule | Rule[Any]] | None = None) → List[Entrance]¶

Connects current region to regions in exit dictionary. Passed region names must exist first.

Parameters:

  • exits – exits from the region. format is {“connecting_region”: “exit_name”}. if a non dict is provided, created entrances will be named “self.name -> connecting_region”

  • rules – rules for the exits from this region. format is {“connecting_region”: rule}

add_locations(locations: Mapping[str, int | None], location_type: type[Location] | None = None) → None¶

Adds locations to the Region object, where location_type is your Location class and locations is a dict of location names to address.

Parameters:

  • locations – dictionary of locations to be created and added to this Region {name: ID}

  • location_type – Location class to be used to create the locations with
